Test timer resolution

This test generates timestamps in a dedicated web worker and compiles a few statistics
It can only test the resolution of timers, not their accuracy.
A test run generates 25 million (25165824) timestamps
Test checks if the clock is monotonic

% of total: the ratio of the number of distinct timestamps to all generated timestamps
Diff is the difference between two consecutive distinct timestamps
diff = timestamp_n+1 - timestamp_n
Here, "Jitter" is absolute difference between two consecutive diffs
jitter = abs((timestamp_n+2 - timestamp_n+1) - (timestamp_n+1 - timestamp_n))

Aside from the standard statistics like the mean and standard deviation of the samples,
this test also calculates a few more robust statistics.
The sampling process can be easily disturbed by a lot of external factors,
many of which can't be controlled from within the test environment.
Things like the load on the system and fore-/background tab/window and how
the browser reacts to those changes can all influence the test result.
The more robust stats ignores outliers and can be a better indicator of the behavior of the clock
